# FlashDraws

## Objetivo do projeto

Nosso objetivo será atender a carêcia do mercado por um aplicativo open source que auxilie em sessões de desenho rápido, ou FlashDraws, como, apartir de aqui, o chamaremos.

## Escopo

Planejamos esse escopo para desenvolvermos o nosso TCC

- **Tipo de Aplicativo**: Desktop Multiplataforma
- **Engine**: [Qt](https://www.qt.io/)
- **Linguagem de programação**: [C++](https://pt.wikipedia.org/wiki/C%2B%2B)
- **Categorias em que o aplicatiivo se encaixa**:
  - Open Source
  - Educacional

## Método *FlashDraws*

Esse é um método de estudo que consiste em, com base em uma referência, a expressar ou copiar da melhor forma possível. Isso é muito feito quanto se quer melhorar em anatomia, desenho gestual ou quanto quer se aquecer antes de desenhar algo.

Pense em [FlashCards](https://en-wikipedia-org.translate.goog/wiki/Flashcard?_x_tr_sl=en&_x_tr_tl=pt&_x_tr_hl=pt&_x_tr_pto=tc) e você terá algo muito parecido com o *FlashDraws*. A principal diferença entre os dois é que no *FlashDraws* o tempo é uma peça fundamental e não existe uma resposta errada.

### termos desse método
- **Sessão**: se refere ao inicio, meio e o fim desse método.

- ***FlashDraw***: um termo inventado por nós para evitar confusão, é o desenho e sua referêcia.
- **Desenho**: é aquilo que você, como usuario, está fazendo.
- **Referêcia**: é aquilo que você está se baseando para fazer o desenho.


## Funcionalidades

- **Configuração**: o usuario decide a quantidade de *FlashDraw* e quanto tempo cada um terá.
- **Refêrencias**: o usuario pode escolher qual pasta própria será usada para buscar as referências.
